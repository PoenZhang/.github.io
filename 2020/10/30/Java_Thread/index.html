1.Thread代表什么？

Q: “Java 创建线程的几种方式” A: “实现Runnable、Callable。线程池、继承Thread” are you ok ?。

通过深入理解JVM这本书，我们知道Java线程模型是一一对应模型，即一个Java线程对应操作系统一个轻量级进程。也就是说，正常new Thread().start()这个动作，会让操作系统中多出一个轻量级进程。所以，应该是new Thread().start() 对应一个线程实体才对，和Runnable、Callable有啥关系？。

这篇文章是我想表达的😂 https://www.linuxidc.com/Linux/2016-03/128997.htm

/**
* Runnable、Callable应该算是约定的任务接口,因为Thread中是执行的Runnable target的run的,
* 实际线程的创建只有new Thread().start0()
* Thread对象在未调用本地方法start0()之前应该是没有在操作系统中创建对应的轻量级进程的
* #这只是我在当前水平下的自我理解,水平有限，见谅。
*/
public class Thread implements Runnable{
    /* What will be run. */
    private Runnable target;

    @Override
    public void run() {
        if (target != null) {
            target.run();
        }
    }
    ........
}

2.Thread 状态变化

线程状态图

  • new Thread() NEW状态 此时只是堆区中多了一个Thread对象，操作系统并没有相应的进程创建

  • thread.start() RUNNABLE状态 操作系统中创建了相应进程，等待cpu调度

  • Running状态 cpu调度执行阶段,即回调执行Thread.run()方法的状态

    • run方法执行完成 -> 则线程正常退出TERMINATED状态
    • run方法中执行以下
      • sleep(long)
      • Object#wait(long) 和对应Object#notify() 方法都必须在synchronized (obj){} 同步区域中才能使用
      • Thread#join(long)
      • LockSupport#parkNanos
      • LockSupport#parkUntil

​ 进入TIMED_WAITING**状态。让当前线程停止一段时间,时间点到则恢复执行 -> RUNNABLE**状态

    • run方法执行以下方法
      • object#wait() 理解: 释放当前线程持有的object对象锁,并将当前线程移动到object的waitting队列中,

此状态的线程需要Object#notify()｜Object#notifyAll() 来显示将线程从相应对象的waiting队列移除。

      • thread.join() 理解: 底层也是wait方法,所以也需要唤醒,只不过这个唤醒是jvm来做的。当thread类执行完成以后,会**notifyAll()**一次，唤醒wait在thred对象上的线程。

参考资料 https://www.jianshu.com/p/fc51be7e5bc0

      • LockSupport#park() 需要对应的**LockSupport#unpark(Thread)解除 WAITING**状态

线程进入WAITING状态。

    • run方法执行
      • Thread.yield() 让出cpu

线程重回**RUNNABLE状态，等待cpu调度

  • BLOCKED状态:线程处于锁对象的sync队列中的状态。

再看下所谓的对象的waiting队列和sync队列。这块涉及到synchronized 锁关键字的解析，此处不做细致解析，简单介绍一下在synchronized 同步块中调用object#wait() 和 o**bject#notify() 的线程变化

图片来源:https://blog.csdn.net/pange1991/article/details/53860651

/**
* 目的:t1线程wait在lock对象上,让t2来notify t1线程
* 问题:如何保证t1和t2执行的有序性。【不用juc包实现】
* 在t2中使用t1.join是有问题。
*/
public class ThreadTest {
    private static Object lock = new Object();
    private static volatile boolean flag = false;
    private static CountDownLatch countDownLatch = new CountDownLatch(1);
    private static Semaphore semaphore = new Semaphore(0);

    public static void main(String[] args) throws InterruptedException {
        Thread t1 = new Thread(new Runnable() {
            @Override
            public void run() {
                synchronized (lock){
                    System.out.println(Thread.currentThread().getName() + " waiting");
                    try {
                        flag = true;
//                        countDownLatch.countDown();
//                        semaphore.release();
                        lock.wait(); //当thread被notify,重新获取锁后,从此处继续执行
                        System.out.println("after waiting");
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                }
                System.out.println("after sync");
            }
        });
        Thread t2 = new Thread(new Runnable() {
            @Override
            public void run() {
                try {
                    //t1.join()问题！join的notify动作在t1线程退出时才会触发,而
                    //t1线程的退出又依赖于t2的notify。所以此处会陷入死锁
                    while (true) {
                        if(!flag){
                          continue;
                        }
//                        countDownLatch.await();
//                        semaphore.acquire();
                        synchronized (lock) {
                            System.out.println(Thread.currentThread().getName() + "notify");
                            lock.notify();

                            System.out.println(Thread.currentThread().getName() + " 等10s再释放锁");
                            Thread.sleep(10000);
                        }
                        break;
                    }
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        });
        //t2先开始,会有什么问题，需要怎么解决？或者说如何保证t1在t2之前执行?
        t2.start();
        t1.start();
    }
}

3.Thread常用api及实现原理

  • thread.interrupt() 中断线程，即设置thread引用的线程中断位为true

  • thread.isInterrupted(boolean) 仅返回****线程的中断状态,且是否清除中断位

    • boolean=true 即thread.interrupted() 清除**当前执行**线程中断位,并返回清除之前线程的中断状态
    • boolean=false 即****thread.isInterrupted() 返回thread线程是否中断

  • thread.isAlive() thread是否处于 start后未终止退出状态

  • thread.join(long) 当前执行线程wait在thread上，等待notify。thread的notify发生在run执行完成退出时。
